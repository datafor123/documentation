# **v8.01 Release Notes**

# Datafor 8.01 Release Notes

## Highlights

### 1. Support for Combined Interactions Across Multiple Charts

Enhanced filtering and interaction linkage among multiple visual components within the same dashboard. Users can now perform linked analysis across multiple charts in a single dashboard, improving cross-chart exploration, filtering, and comparison.

### 2. Enhanced Aggregation Table and Modeling Capabilities

Improved both the frontend and backend logic for aggregation tables.

Reference: https://help.datafor.com.cn/documentation/Model/Use-Aggregation-Tables/

### 3. Enhanced SDK Embedding Capabilities

Supports calling Datafor pages through the SDK, using JWT authentication and session reuse, further improving the integration experience for embedded analytics scenarios.

Reference: https://help.datafor.com.cn/documentation/SDK-Embedding/

## Feature Enhancements and UX Improvements

- Added color assignment for multiple measures in charts.
- Added zoom interactions in report edit mode.
- Added component position locking for easier page layout maintenance.
- Adjusted the component menu and removed unnecessary options.
- Improved Tip interactions, including support for closing and delayed display.


## Performance and Stability Improvements

- Introduced a unified service proxy for AI requests, improving cookie handling and CSP cross-origin compatibility to enhance overall system consistency.
- Added a native caching mechanism and cache compression strategy to improve multidimensional analysis and SQL query performance.
- Added and strengthened multiple persistent caching capabilities for multidimensional analysis and aggregations, improving overall response speed and stability.
- Optimized model compression algorithms, multi-table aggregation, and automatic inheritance of aggregation table links to reduce exceptions and resource consumption.
- Enhanced performance monitoring and logging capabilities for easier issue diagnosis and system health monitoring.

## AI Performance Enhancements and Bug Fixes

- Optimized the AI request pipeline and service proxy mechanism to improve the response speed and stability of AI calls.

## Important Bug Fixes

- Fixed issues where map conditional colors did not take effect, and where values reverted to defaults after adjustment while the map data did not update.
- Fixed critical issues such as null pointer exceptions in multi-table joins, RowCount calculation errors, and NoLink path logic errors.
- Fixed multiple issues affecting stability, including user login, permission validation, and external API parameters.
- Fixed occasional errors related to Pentaho integration and JCR queries.
